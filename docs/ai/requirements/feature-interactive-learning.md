# Requirements & Problem Understanding

## Problem Statement
**What problem are we solving?**
- After migrating to the real MongoDB backend, the React frontend lost its interactive **Exercises** and contextual **Sentences** because these models weren't part of the initial data ingestion.
- The app currently lacks a **User Authentication** system (Login/Register) and a way to save **Learning Progress**. Without this, users cannot track which lessons they have completed, nor can they personalize their flashcard review sessions based on spaced repetition.

**Who is affected by this problem?**
- Vietnamese learners of Traditional Chinese who need context (sentences), active recall practice (exercises), and motivation (progress tracking).

## Goals & Objectives
**What do we want to achieve?**
- **Primary goals:**
  - Design and implement MongoDB schemas for `Sentence` and `Exercise` models.
  - Implement a basic User Authentication flow (JWT-based Login/Registration).
  - Implement a `UserProgress` tracking system to record completed lessons and mastery levels.
- **Secondary goals:**
  - Seed the database with sample sentences and generated exercises linked to the existing vocabulary.
- **Non-goals:**
  - Advanced spaced repetition algorithms (SRS) are out of scope for *this* phase; we just need basic tracking first.
  - Social login (Google, Facebook) is out of scope; standard email/password is sufficient.

## User Stories & Use Cases
**How will users interact with the solution?**
- As a learner, I want to create an account and log in securely so my data is preserved.
- As a learner, I want to see example sentences for new vocabulary words so I know how to use them.
- As a learner, I want to take interactive quizzes (multiple-choice) at the end of a lesson to test my knowledge.
- As a learner, I want to see a visual indicator of which lessons I have already finished.

## Success Criteria
**How will we know when we're done?**
- A user can register, log in, and receive an auth token.
- The `Lesson` API endpoint returns nested `sentences` and `exercises`.
- The frontend `ReviewPage`, `ProgressPage`, and `HomePage` operate on real user-specific data from the backend.
- A progress API endpoint allows saving a completed lesson metric.

## Constraints & Assumptions
**What limitations do we need to work within?**
- **Data Constraints:** We'll need to generate or map sample sentences. We can auto-generate basic multiple-choice exercises using script logic during ingestion.
- **Technical Constraints:** Must integrate smoothly with the existing Node.js/Mongoose stack and Vite React frontend (using Zustand).
- **Security:** Passwords must be hashed (e.g., bcrypt).

## Questions & Open Items
**What do we still need to clarify?**
- Should exercises be hardcoded in the database, or dynamically generated by the backend API per lesson request? *(Recommendation: Store them in the DB for manual curation later, but write an ingestion script to auto-generate them initially).*
